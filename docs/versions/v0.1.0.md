# OpenKATE v0.1.0：工程与项目底座

> 前置版本：无
> 预计工期：3 个 Sprint
> 版本类型：内部 Alpha

## 1. 版本目标

建立多服务研发、部署、数据隔离和可观测性基线，交付最小可用的工作区、项目、环境和成员管理。该版本不实现验证场景和真实执行。

## 2. 用户需求

| ID | 需求 | 验收结果 |
| --- | --- | --- |
| `FND-001` | 用户可以登录并进入工作区 | 未认证请求被拒绝，登录后可读取用户信息 |
| `FND-002` | Owner 可以创建和编辑项目 | 项目创建后出现在项目列表 |
| `FND-003` | Maintainer 可以管理测试环境 | 环境包含名称、Base URL 和安全策略 |
| `FND-004` | Owner 可以邀请成员并分配角色 | Viewer 无法执行写操作 |
| `FND-005` | 用户可以查看平台服务状态 | 健康页展示核心模块可用性和降级状态 |
| `FND-006` | 管理操作可以审计 | 项目、环境和权限变更可按操作者查询 |

## 3. 服务实现

| 模块 | 实现内容 |
| --- | --- |
| `gateway-service` | JWT/OIDC、RBAC、Request ID、限流、统一错误、OpenAPI 聚合 |
| `project-service` | Workspace、Project、Environment、Member、Role、Policy 聚合 |
| 其他 8 个领域服务 | 独立 FastAPI 应用、健康检查、指标、Schema、空能力注册 |
| `executor-ui/api/state` | 独立 Worker 骨架、能力注册和心跳，不实现真实执行 |
| `apps/web` | 登录、工作区切换、项目列表、环境和成员页面 |

## 4. 工程与基础设施

- 建立 `uv Workspace`、`pnpm Workspace`、Makefile。
- 创建 1 个 Gateway、9 个领域服务和 3 个 Worker 的独立构建入口。
- Docker Compose Profile：`core`、`ai`、`executors`、`observability`。
- PostgreSQL：每服务独立 Schema、Migration User 和 Runtime User。
- Temporal：Namespace、Task Queue、最小 Workflow 示例。
- NATS JetStream：Stream、Consumer、重试和死信策略。
- MinIO：Bucket、项目路径和预签名 URL。
- OpenTelemetry：Trace、Metric 和结构化日志。
- Python 统一 Ruff、mypy、pytest、Pydantic；Web 统一 TypeScript strict、ESLint、Vitest。
- 实现 Outbox Publisher 和幂等 Consumer 基础包。

## 8. 前端范围

- 登录和未授权页面。
- 工作区与项目切换。
- 项目创建、编辑和归档。
- 环境列表与编辑。
- 成员列表、角色调整和移除。
- 系统健康与降级状态。
- Loading、Empty、Error、Permission Denied 状态。

## 9. 测试计划

- 角色权限矩阵单元和 API 集成测试。
- 服务账号无法读取其他 Schema 的数据库权限测试。
- 重复事件只产生一次业务效果的幂等测试。
- Migration 空库升级和前一版本回滚测试。
- 任一空服务停止时，其他服务保持可用并报告降级。
- OpenAPI 生成 TypeScript Client 并通过前端编译。

## 10. 版本验收

- 一条命令可以启动全部模块。
- 每个服务可以独立构建、测试和发布。
- 新开发者 30 分钟内完成本地首次启动。
- Owner 创建项目、环境、成员并在审计日志中看到全部操作。
- `project.created.v1` 从 Outbox 经 NATS 到消费者完成幂等处理。

## 11. 明确不包含

- 验证场景、执行器真实运行、Agent、报告和 Connector。
