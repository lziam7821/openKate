# OpenKATE v0.2.0：Validation Center

> 前置版本：v0.1.0
> 预计工期：2 个 Sprint
> 版本类型：Product Alpha

## 1. 版本目标

建立业务验证场景的手工生命周期，使平台在没有 AI 和执行器的情况下也能管理可版本化、可 Review 的验证资产。

## 2. 用户需求

| ID | 需求 | 验收结果 |
| --- | --- | --- |
| `VAL-001` | QA 可以创建验证场景 | 包含业务目标、参与者、前置条件和风险级别 |
| `VAL-002` | QA 可以维护风险和业务不变量 | 风险与场景版本绑定 |
| `VAL-003` | QA 可以定义 UI/API/State 证据点 | 每个证据点包含观察目标和断言 |
| `VAL-004` | Reviewer 可以提出修改意见 | Review 记录作者、内容和处理状态 |
| `VAL-005` | Reviewer 可以批准或拒绝场景 | 未批准场景不能进入正式运行 |
| `VAL-006` | 用户可以查看场景版本差异 | 任意两个版本可比较 |
| `VAL-007` | 用户可以搜索和筛选场景 | 支持状态、风险、标签和 Owner |

## 3. 领域状态机

```text
draft -> in_review -> approved -> archived
             |            |
             v            v
          rejected     deprecated
```

- `draft` 可以编辑。
- `in_review` 只允许 Review 相关操作。
- `approved` 只能创建新版本，不能原地修改。
- `rejected` 修改后生成新版本并重新提交。

## 4. 服务实现

| 模块 | 实现内容 |
| --- | --- |
| `validation-service` | Scenario Aggregate、版本状态机、Risk、EvidencePoint、Review |
| `gateway-service` | 场景路由、RBAC、乐观锁冲突映射 |
| `report-service` | 场景列表 Read Model 和筛选索引 |
| `apps/web` | 场景列表、创建向导、详情、证据编辑、Review 和版本 Diff |

## 8. 前端范围

- 验证场景列表、搜索、筛选和分页。
- 场景创建与编辑向导。
- 业务目标、前置条件、风险、不变量和证据计划编辑。
- Review 评论、问题处理和审批。
- 场景版本历史和字段级 Diff。
- 并发编辑冲突提示和重新加载。

## 9. 测试计划

- 状态机合法与非法迁移测试。
- 两人同时编辑时的乐观锁冲突测试。
- 场景、风险、证据的版本一致性测试。
- Viewer、Developer、Reviewer、Maintainer 权限测试。
- Read Model 事件重复消费和重建测试。

## 10. 版本验收

- 手工创建 `checkout` 场景。
- 完成提交、Review、拒绝、修改、再次提交和批准流程。
- Approved 版本不能被原地修改。
- 场景详情可以追踪操作者、版本和审计记录。
- 场景列表在 `report-service` 暂时不可用时明确展示降级状态。

## 11. 明确不包含

- 真实执行、自动报告、AI 生成和知识召回。
