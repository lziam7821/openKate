# OpenKATE v0.5.0：AI Scenario Generation

> 前置版本：v0.4.0
> 预计工期：3 个 Sprint
> 版本类型：AI Beta

## 1. 版本目标

从 PRD 和 OpenAPI 自动生成结构化、可追溯、必须经人工 Review 的验证场景，并接入已有真实执行闭环。

## 2. 用户需求

| ID | 需求 | 验收结果 |
| --- | --- | --- |
| `AI-001` | 用户可以上传 Markdown/PDF PRD | 原始文件和解析结果均可查看 |
| `AI-002` | 用户可以导入 OpenAPI | 接口、Schema、鉴权和错误码被结构化 |
| `AI-003` | 用户可以发起场景生成 | 任务展示每个 Agent 阶段状态 |
| `AI-004` | 生成内容包含来源引用 | 关键目标和规则可回到原文位置 |
| `AI-005` | AI 可以生成风险和证据计划 | 输出通过 Schema 和执行器能力校验 |
| `AI-006` | Review Agent 可以指出缺失和幻觉 | 问题按严重级别展示 |
| `AI-007` | 人工可以接受、修改或拒绝输出 | AI 输出不能直接进入 Approved |
| `AI-008` | 管理员可以查看模型成本和版本 | 记录 Model、Prompt、Token 和 Latency |

## 3. Agent Workflow

```text
Asset Parse
  -> Requirement Agent
  -> Intent Agent
  -> Scenario Planner
  -> Risk Model Agent
  -> Evidence Planner
  -> Review Agent
  -> Human Review
  -> Validation Scenario Version
```

Human Review 使用 Temporal Signal 暂停和恢复 Workflow。Agent 每一步保存结构化输出，不依赖进程内会话状态。

## 4. 服务实现

| 模块 | 实现内容 |
| --- | --- |
| `asset-service` | Markdown/PDF/OpenAPI 上传、解析、版本和来源定位 |
| `agent-service` | LLM Gateway、Provider Adapter、Prompt Version、结构化输出和成本统计 |
| `workflow-service` | `ScenarioGenerationWorkflow`、Review Signal、恢复和重试 |
| `validation-service` | AI Draft 导入、来源引用和人工修改差异 |
| `project-service` | 项目模型策略、预算和脱敏策略 |
| `apps/web` | 资产上传、生成进度、来源对照、AI Review 和确认页面 |

## 8. 质量与安全门禁

- Agent 输出必须通过 Pydantic Schema。
- 来源不存在时标记为 `inferred`，不能伪造引用。
- 高风险场景必须人工审批。
- 生成质量低于 `0.75` 自动进入需修改状态。
- Prompt、模型、规则变更必须运行固定评测集。
- 上传内容和模型输入执行恶意指令隔离与敏感字段脱敏。
- 模型不可用时任务进入 `waiting_model`，不生成空结果。

## 9. 测试计划

- 20 份固定 PRD/OpenAPI 样本作为初始评测集。
- Asset Parser 文本位置和引用稳定性测试。
- 每个 Agent 的 Schema、超时、重试和恢复测试。
- Human Review 暂停超过 24 小时后的恢复测试。
- Provider 切换后输出 Contract 一致性测试。
- Prompt Injection 和敏感字段泄漏测试。

## 10. 版本验收

- 业务目标提取准确率 >= 85%。
- 输出 Schema 通过率 >= 95%。
- 不存在页面、接口和字段的幻觉率 <= 5%。
- 至少 70% Evidence Point 可转换为有效执行步骤。
- 从样例 PRD 生成 `checkout` 场景，人工批准后完成真实运行和报告。
- 每个生成字段都能显示来源或 `inferred` 标识。

## 11. 明确不包含

- 历史知识 RAG、BadCase 规则自动召回和 PR Diff 影响分析。
