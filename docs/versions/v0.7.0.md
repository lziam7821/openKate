# OpenKATE v0.7.0：CI/CD & Connectors

> 前置版本：v0.6.0
> 预计工期：2 个 Sprint
> 版本类型：Workflow Beta

## 1. 版本目标

将 OpenKATE 接入研发流程，让代码变更自动触发受影响业务场景并将结果回写协作系统。

## 2. 用户需求

| ID | 需求 | 验收结果 |
| --- | --- | --- |
| `CICD-001` | 管理员可以连接 GitHub/GitLab | Webhook 验签并同步仓库信息 |
| `CICD-002` | PR 可以触发验证任务 | 运行记录关联 Commit 和 PR |
| `CICD-003` | 系统可以分析 Diff 影响 | 选择关联页面、接口、对象和场景 |
| `CICD-004` | CI 可以读取状态和 JUnit | 流水线可按策略阻断 |
| `CICD-005` | 结果可以通知 Feishu/Slack | 通知包含结论和报告链接 |
| `CICD-006` | 失败可以创建或关联 Jira Issue | Issue 保留 Run、Evidence 和 Commit 关系 |
| `CICD-007` | 管理员可以配置阻断策略 | failed、inconclusive、高风险可分别配置 |

## 3. 服务实现

| 模块 | 实现内容 |
| --- | --- |
| `connector-service` | GitHub、GitLab、Jira、Feishu、Slack Adapter 和同步游标 |
| `asset-service` | Code Diff 解析、文件/函数/接口影响对象 |
| `agent-service` | Diff Requirement Agent 和影响候选排序 |
| `workflow-service` | `PullRequestValidationWorkflow` |
| `validation-service` | 场景与代码、接口、页面和业务对象关联 |
| `report-service` | PR Check Summary、JUnit 和通知 Read Model |
| `project-service` | Connector 配置、触发规则和阻断策略 |

## 4. Workflow

```text
Receive Signed Webhook
  -> Normalize Pull Request Event
  -> Fetch and Parse Diff
  -> Calculate Impacted Objects
  -> Select Approved Scenarios
  -> Execute Scenarios
  -> Aggregate Result
  -> Update Commit Status
  -> Notify Team / Link Issue
```

无法可靠识别影响范围时进入人工确认或运行项目配置的安全回退集，不能静默跳过。

## 8. 安全约束

- Webhook 必须验签并防止重放。
- Connector Token 保存到 Secrets Vault，只存 Secret 引用。
- 外部 API 调用使用独立限流、超时、重试和熔断。
- PR 内容视为不可信输入，进入 Agent 前隔离指令和脱敏。
- Connector 权限按项目隔离。

## 9. 测试计划

- Webhook 签名、重放攻击和重复事件测试。
- Connector 限流、分页、断点续传和 Token 失效测试。
- Diff 影响分析固定数据集测试。
- Commit Status 和 Jira/通知 Adapter Contract Test。
- Connector 故障时手工运行和历史报告可用性测试。

## 10. 版本验收

- GitHub/GitLab PR 自动触发受影响场景。
- 结果回写 Commit Status 并提供报告链接。
- CI 下载并解析 JUnit。
- 一个失败运行能够创建 Jira Issue 并关联证据。
- 无法判定影响范围时执行安全回退策略。
- 外部 Connector 停止时平台核心功能不受影响。

## 11. 明确不包含

- Mobile、GraphQL/gRPC、MQ/Log/Trace、Performance 和 Security 通道。
