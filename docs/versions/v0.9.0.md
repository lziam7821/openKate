# OpenKATE v0.9.0：Production Readiness

> 前置版本：v0.8.0
> 预计工期：3 个 Sprint
> 版本类型：Release Candidate

## 1. 版本目标

完成生产部署需要的稳定性、安全性、隔离、扩缩容、备份、升级和运维能力。

## 2. 用户需求

| ID | 需求 | 验收结果 |
| --- | --- | --- |
| `OPS-001` | 管理员可以管理 Secrets 引用 | 页面和 API 永不返回明文 Secret |
| `OPS-002` | 管理员可以查看服务和队列容量 | Worker、Task Queue、Event Lag 可观测 |
| `OPS-003` | 管理员可以配置配额 | 项目并发、Token、存储和运行时间受控 |
| `OPS-004` | 系统支持备份和恢复 | PostgreSQL、Object Storage 和配置可恢复 |
| `OPS-005` | 系统支持滚动升级和回滚 | 服务和 Contract 兼容升级 |
| `OPS-006` | 用户可以查看完整审计记录 | 跨服务操作按 Trace ID 聚合 |
| `OPS-007` | 平台支持数据保留和删除策略 | 到期证据自动归档或删除 |

## 3. 平台实现

- Kubernetes Helm Chart、NetworkPolicy、Pod Security 和 Resource Quota。
- 独立 Service Account、最小权限和 Workload Identity。
- Secrets Vault Adapter、短期凭据和 Secret Rotation。
- PostgreSQL HA、自动备份和 PITR。
- Object Storage Lifecycle、Versioning 和不可变审计存储。
- Temporal Worker Versioning、NATS Consumer 监控和死信重放工具。
- Gateway、Agent、Execution、Report 和 Worker Pool HPA。
- OpenTelemetry Dashboard、SLO、Alert 和 Runbook。
- API/Event/Database 兼容升级自动检查。
- SBOM、镜像签名、依赖扫描、SAST 和 DAST。

## 4. 服务改动

| 模块 | 实现内容 |
| --- | --- |
| `gateway-service` | 全局配额、熔断、审计入口、灰度路由 |
| `project-service` | Quota、Retention Policy、Secret Reference |
| `workflow-service` | Worker Versioning、Dead-letter Recovery、运维 Workflow |
| `execution-service` | 并发控制、资源预算、异常租约回收 |
| `report-service` | 审计聚合、冷热数据和大报告分页 |
| 全部服务 | Graceful Shutdown、Readiness、SLO、资源限制和升级兼容 |
| `apps/web` | 运维概览、配额、Secrets、审计和保留策略页面 |

## 5. 非功能目标

| 指标 | 目标 |
| --- | --- |
| Gateway Availability | >= 99.9% |
| 任务状态丢失 | 0 |
| Event 重复产生业务副作用 | 0 |
| API P95 | <= 500ms，不含长任务 |
| 报告生成 P95 | <= 30s |
| RPO | <= 15min |
| RTO | <= 2h |
| Secret 明文泄漏 | 0 |

## 6. 安全要求

- 项目、Environment、Secret、Object Storage 和 Worker 均强制租户隔离。
- Production Environment 默认只读，写操作需要审批和 Allowlist。
- Prompt、PRD、Log、Request/Response 在模型调用前脱敏。
- 所有镜像使用非 Root 用户、只读文件系统和最小 Capability。
- 对象下载使用短期预签名 URL，并验证项目权限。
- 审计事件写入不可变存储并校验完整性。

## 8. 测试计划

- Worker、Service、NATS、Temporal 和 PostgreSQL 故障注入。
- 滚动升级期间长 Workflow 不丢失。
- 备份恢复和数据一致性校验。
- 100 个并发场景容量测试。
- 项目权限和数据隔离渗透测试。
- 24 小时稳定性运行和资源泄漏检查。
- Secret 扫描和日志脱敏回归。

## 9. 版本验收

- 全新安装、升级、回滚和备份恢复演练通过。
- 单个 Worker Pool 故障不影响控制面。
- 长 Workflow 跨两个 Worker 版本继续执行。
- 达到全部非功能目标。
- 阻断级安全问题为 0。
- 生产 Runbook 能够指导值班人员处理积压、Worker 故障和恢复。

## 10. 明确不包含

- `/api/v1`、Event v1 和 Executor SDK v1 的最终冻结；该工作在 v1.0.0 完成。
