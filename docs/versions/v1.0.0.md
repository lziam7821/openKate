# OpenKATE v1.0.0：General Availability

> 前置版本：v0.9.0
> 预计工期：2 个 Sprint
> 版本类型：GA

## 1. 版本目标

冻结首个稳定产品契约，完成安装、升级、运维、开发和用户文档，形成可公开发布和持续维护的开源版本。

## 2. 用户需求

| ID | 需求 | 验收结果 |
| --- | --- | --- |
| `GA-001` | 新用户可以快速完成安装 | 30 分钟内运行示例场景 |
| `GA-002` | 用户可以完成全业务闭环 | 导入、生成、Review、执行、报告、BadCase、规则发布 |
| `GA-003` | 管理员可以无损升级 | 从 v0.9.x 升级并可回滚 |
| `GA-004` | 开发者可以扩展 Executor | 使用 SDK 和 Contract Test Kit 完成插件 |
| `GA-005` | API/SDK/CLI 有稳定兼容承诺 | 发布版本、文档和兼容矩阵 |
| `GA-006` | 所有产品状态完整 | Loading、Empty、Error、Permission、Degraded 均可用 |

## 3. 产品交付

- Quick Start、完整安装、升级、运维和排错文档。
- 中文与英文核心用户文档。
- `checkout`、`refund`、`inventory` 三个示例场景。
- 示例 Web、API、PostgreSQL、MQ、Mock 和失败 BadCase。
- 稳定的 Web Console 信息架构和错误体验。
- Release Note、Known Limitations 和 Support Policy。

## 4. 技术实现

- 冻结 `/api/v1`、Event v1 和 Executor SDK v1。
- 发布 Docker Compose、Helm、示例配置和升级工具。
- 发布 Python SDK、CLI 和生成的 TypeScript API Client。
- 建立 Migration Guide 和 Compatibility Matrix。
- 发布 Executor/Connector 开发文档和 Contract Test Kit。
- 删除实验性开关和本项目版本演进产生的废弃 Contract。
- 建立发布签名、镜像 Provenance 和可复现构建。

## 5. 稳定契约

```text
HTTP API: /api/v1
Event Contract: *.v1
Executor SDK: 1.x
Python SDK: 1.x
CLI: 1.x
TypeScript API Client: generated from /api/v1
```

- Minor 版本只添加向后兼容能力。
- Deprecated 能力至少保留两个 Minor 版本。
- Security Fix 可以发布 Patch 版本。
- Breaking Change 只能进入下一个 Major。

## 7. 测试计划

- v0.1 至 v0.9 全部核心验收场景回归。
- API、Event 和 Executor Contract 兼容性测试。
- 全新安装、升级、回滚和备份恢复演练。
- Python SDK、CLI 和 TypeScript Client 示例测试。
- Quick Start 在干净环境中计时验证。
- 核心 Agent 评测集和三条示例业务闭环。
- 安全审计、License、SBOM 和依赖漏洞检查。

## 8. GA 发布门禁

- 阻断级安全问题为 0，高风险问题已处置或明确接受。
- API/Event/Executor Contract 兼容性通过。
- `checkout`、`refund`、`inventory` 完成全闭环演示。
- Quick Start 在 30 分钟内成功。
- 升级、回滚、备份恢复全部通过。
- 已知限制、容量边界和运维风险全部文档化。
- 所有发布 Artifact 已签名并可验证。

## 9. 发布后策略

- `v1.0.x`：只修复缺陷和安全问题。
- `v1.1.0`：兼容性功能增强和社区 Executor。
- `v1.2.0`：多租户运营、成本和企业集成增强。
- `v2.0.0`：仅在稳定 Contract 确需破坏性演进时规划。

## 10. 明确不包含

- 未经验证的实验性 Agent 自主发布规则。
- 默认允许生产环境写操作。
- 未进入兼容性矩阵的第三方 Executor 和 Connector。
